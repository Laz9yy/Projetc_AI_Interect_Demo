# 🎯 AI 虚拟人设陪伴交互页 — 项目规划书

> **Vibe Coding 风格** · 轻量化 2D 虚拟人情感陪伴 Web 应用  
> 主打视觉氛围感与趣味交互，无复杂业务逻辑

---

## 一、项目定位

| 维度 | 内容 |
|------|------|
| **产品名** | **心言 · MindChat**（暂定） |
| **定位** | 轻量化 2D 虚拟人情感陪伴 Web 应用 |
| **目标用户** | Z 世代年轻人，寻求情绪树洞、轻陪伴 |
| **核心 slogan** | *你的专属虚拟灵魂搭子* |
| **视觉基调** | 毛玻璃 / 渐变噪点 / 柔光 / 圆润卡片 / 微动效 / 深色 + 柔和紫粉调 |
| **交互调性** | 松弛、治愈、有温度 — 每个微交互都有"呼吸感" |
| **核心差异化** | 不堆 3D 模型，靠 2D 角色 + 丰富微表情 + 沉浸氛围实现陪伴感 |

### 1.1 功能概览

- 自定义人物性格与外貌
- 2D 虚拟形象实时对话
- 自动生成动态表情
- 轻量化聊天交互

### 1.2 项目意义

- 打造轻量化情绪陪伴载体，满足年轻人陪伴、树洞倾诉需求
- 无需部署 3D 大模型，轻量化方案成本低
- 调用通用对话模型 + 开源卡通素材即可快速落地

## 二、技术选型与架构设计

`
┌─────────────────────────────────────────────────┐
│                  Frontend                        │
│  Vite + React 18 + TypeScript                   │
│  TailwindCSS + Framer Motion                    │
│  Zustand (状态管理)                              │
│  Canvas / PixiJS (2D 形象渲染层)                 │
├─────────────────────────────────────────────────┤
│                   APIs                           │
│  OpenAI-compatible Chat API (对话生成)          │
│  自建轻量表情规则引擎 (expression engine)       │
├─────────────────────────────────────────────────┤
│                 Assets                           │
│  SVG 组件化角色 (可换装 / 换色)                  │
│  CSS 粒子 / 光晕 / 波纹背景特效                 │
│  Tone.js (环境音效, 可选)                       │
└─────────────────────────────────────────────────┘
`

### 2.1 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| **Vite + React** | 启动快、HMR 即时、生态成熟，适合 Vibe Coding 快速迭代 |
| **TailwindCSS** | 零设计系统心智负担，Utility-First 快速出 UI |
| **Framer Motion** | 声明式动画 API，轻松实现呼吸、淡入、弹性等氛围感动效 |
| **PixiJS / 纯 SVG** | 2D 形象不需要 3D 引擎，轻量高性能 |
| **Zustand** | 比 Redux 轻得多，角色状态、对话历史管理恰到好处 |

## 三、功能模块拆分

### M1 — 角色创建工坊

- 性格标签选择系统（可选：外向 / 内向 / 幽默 / 温柔 / 毒舌 等）
- 外貌自定义：发型、发色、眼睛、服装（SVG 组件组合）
- 角色命名 + 简短背景设定输入
- **✨ 氛围亮点**：创建时角色随选择实时变化，伴有柔和粒子飘散

### M2 — 2D 虚拟形象渲染

- SVG 组件化角色 + Canvas 合成
- 基础 idle 动画（呼吸、眨眼）
- 6–8 组动态表情映射（开心、思考、难过、惊讶、害羞、生气、疑惑、放松）
- **✨ 氛围亮点**：角色有"微表情"过渡，不是生硬切换

### M3 — 轻量化聊天交互

- 聊天气泡（圆润、渐变边框、打字机逐字效果）
- 角色头像实时同步表情
- 快速回复建议（2–3 个预设选项浮球）
- 聊天记忆（当前会话上下文）
- **✨ 氛围亮点**：对话间有柔和过渡动效，气泡像"浮起来"的

### M4 — 表情与情绪引擎

- 基于 LLM 回复内容自动匹配表情
- 规则引擎：关键词 → 情绪标签 → 表情动画
- 表情过渡动画（ease-out 缓出，不跳变）
- **✨ 氛围亮点**：表情会"慢慢浮现"而不是瞬切

### M5 — 氛围增强系统

- 动态背景（渐变流动 / 粒子星云 / 极光效果）
- 环境音效（雨声、篝火、海浪 — 可选开关）
- 主题色随情绪变化（开心 → 暖橙、平静 → 蓝紫）
- **✨ 氛围亮点**：整个页面是"活的"，背景在缓缓流动

## 七、Vibe Coding 氛围感策略

> 这是整个项目的灵魂 — **氛围优先，交互其次，功能最后**。

### 7.1 色彩策略

- **主色渐变**：#7C5CFC（紫罗兰） → #FF6B9D（粉红）
- **背景底色**：#0A0A1A（深邃夜空）
- **辅助色**：#00D4FF（电蓝，用于高亮和发送按钮）
- **情绪主题联动**：
  - 开心 → 暖橙 #FF9A56
  - 平静 → 蓝紫 #7C5CFC
  - 难过 → 冷蓝 #4A90D9
  - 浪漫 → 粉红 #FF6B9D

### 7.2 玻璃拟态

- 卡片使用 ackdrop-filter: blur(20px) + 半透明边框
- 叠加 ox-shadow 内发光效果
- 圆角统一 16px–24px

### 7.3 微动效矩阵

| 交互点 | 动画效果 |
|--------|----------|
| 页面进入 | 角色淡入 + 背景渐显（1.5s ease-out） |
| 角色说话 | 呼吸频率加快 + 表情同步 |
| 消息发送 | 气泡弹性弹出（spring） |
| 快捷回复 | 浮球逐个淡入上浮 |
| 角色创建 | 选择标签时粒子飘散 |
| 主题切换 | 背景色平滑过渡 |
| Hover 效果 | 卡片轻微上浮 + 边框发光 |

### 7.4 氛围层叠

`
Z-Index 层级
├── 10: UI 交互层（按钮、输入框）
├── 9:  玻璃卡片层
├── 5:  角色形象层 (Canvas)
├── 3:  前景粒子层
├── 2:  极光/光晕层
└── 1:  渐变流动背景层
`

### 7.5 角色"活"的感觉

- 空闲时自动呼吸（SVG 缩放 + 平移循环）
- 随机眨眼（每 3–5s 一次）
- 轻微头部晃动（每 8–12s 一次）
- 不对话时也会随机出现微表情变化

### 7.6 音景设计

- 可选环境白噪音：雨声、篝火、海浪、森林
- 消息发送提示音（轻柔，不打扰）
- 角色表情变化时的细微音效

### 7.7 加载即体验

- Loading 界面直接用角色动画 + 渐入
- 骨架屏使用毛玻璃风格，与主题统一
- 不让用户看到白屏或空白

## 八、扩展可能性

- **多角色切换**：预设多套虚拟人设，一键切换
- **对话记忆持久化**：接入 localStorage 或轻量后端
- **角色语音输出**：接入 TTS API，角色"开口说话"
- **每日心理签到**：结合情绪记录，生成每周情绪曲线
- **角色养成**：陪伴时长解锁新外观 / 新性格特质
